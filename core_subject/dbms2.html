Term / Topic / Concept Explanation (Easy to Remember)
Database System Organized collection of related data stored and accessed electronically. Used to store, retrieve, and manage data efficiently.
Purpose of DBMS To eliminate data redundancy, maintain data consistency, provide data security, enable data sharing, and ensure data integrity.
Data Independence Ability to change database schema at one level without affecting other levels. Two types: Logical (change conceptual schema) and Physical (change storage).
Relational Database Database that stores data in tables (relations) with rows and columns. Data is related through common fields (keys).
Database Language DDL (Data Definition Language) - defines structure, DML (Data Manipulation Language) - manipulates data, DCL (Data Control Language) - controls access.
Database System Architecture Three levels: External (user view), Conceptual (logical structure), Internal (physical storage). Provides abstraction and independence.
Database Users End users (naive, sophisticated), Application programmers, Database Administrators (DBA).
DBA (Database Administrator) Person responsible for managing database - creating schemas, granting access, backup/recovery, performance monitoring.
Data Models Collection of concepts to describe database structure. Types: Hierarchical, Network, Relational, Object-oriented.
Importance of Data Models Provides blueprint for database design, helps in communication, ensures proper organization, reduces complexity.
Building Blocks of Data Model Entities (objects), Attributes (properties), Relationships (connections), Constraints (rules).
Degrees of Data Abstraction Three levels: Physical (how data is stored), Logical (what data is stored), View (how users see data).
ER Model Entity-Relationship Model - graphical representation of database structure using entities, attributes, and relationships.
Entity Real-world object with independent existence (e.g., Student, Course, Employee).
Attribute Property or characteristic of an entity (e.g., Student has Name, Roll No, Age). Types: Simple, Composite, Derived, Multi-valued.
Relationship Association between entities (e.g., Student enrolls in Course). Types: One-to-One, One-to-Many, Many-to-Many.
Constraints in ER Model Rules that restrict data: Cardinality (1:1, 1:N, M:N), Participation (Total - all must participate, Partial - optional).
ER Diagram (ERD) Visual representation using symbols: Rectangle (Entity), Ellipse (Attribute), Diamond (Relationship), Lines (connections).
Weak Entity Set Entity that cannot exist without another entity (owner entity). Shown with double rectangle. Has partial key, needs owner's key to identify.
Term / Topic / Concept Explanation (Easy to Remember)
Relational Model Structure Data organized in tables (relations). Each table has rows (tuples/records) and columns (attributes/fields).
Domain Set of allowed values for an attribute (e.g., Age domain: 0-100, Gender domain: M/F).
Relation Table with rows and columns. Each row is a tuple, each column is an attribute. Must have unique name.
Relational Algebra Procedural query language with operations to manipulate relations. Foundation for SQL queries.
Selection (σ) Selects rows that satisfy a condition. σ(condition)(Table). Example: σ(age>20)(Student) - selects students older than 20.
Projection (π) Selects specific columns from table. π(columns)(Table). Example: π(name,age)(Student) - shows only name and age.
Union (∪) Combines tuples from two tables, removes duplicates. Tables must be union-compatible (same attributes).
Set Difference (−) Returns tuples in first relation but not in second. A − B gives rows in A but not in B.
Cartesian Product (×) Combines every row of first table with every row of second table. Creates all possible combinations.
Rename (ρ) Renames relation or attributes. ρ(newname)(Table) or ρ(newname(attr1,attr2))(Table).
Join Operations Combines related tuples from two tables. Types: Natural Join, Theta Join, Equi Join, Outer Join (Left, Right, Full).
Division Finds tuples in one relation that match all tuples in another. Used for 'for all' queries.
Tuple Relational Calculus Nonprocedural query language. Specifies what to retrieve, not how. Uses predicates and quantifiers.
Codd's Rules 12 rules for true relational database: Rule 0 (foundation rule), information rule, guaranteed access, null values, dynamic catalog, etc.
UML (Unified Modeling Language) Standard notation for visualizing system design. Includes class diagrams similar to ER diagrams.
Keys in Relational Model Super key (uniquely identifies), Candidate key (minimal super key), Primary key (chosen candidate), Foreign key (references another table).
Integrity Rules Entity Integrity (primary key cannot be null), Referential Integrity (foreign key must match primary key or be null).
Normalization Process of organizing data to reduce redundancy and improve integrity. Done through normal forms.
1NF (First Normal Form) Each column has atomic (indivisible) values. No repeating groups. Each row is unique.
2NF (Second Normal Form) Must be in 1NF. No partial dependency (non-key attribute depends on part of composite key).
3NF (Third Normal Form) Must be in 2NF. No transitive dependency (non-key attribute depends on another non-key attribute).
BCNF (Boyce-Codd Normal Form) Stronger version of 3NF. For every dependency A→B, A must be a super key.
Multi-valued Dependency When one attribute determines multiple independent sets of values. A ↠ B means A multi-determines B.
4NF (Fourth Normal Form) Must be in BCNF. No multi-valued dependencies. Each multi-valued fact recorded in separate table.
Term / Topic / Concept Explanation (Easy to Remember)
SQL (Structured Query Language) Standard language for relational databases. Used to create, query, update, and manage databases.
Data Definition Language (DDL) SQL commands to define database structure: CREATE, ALTER, DROP, TRUNCATE.
CREATE TABLE Creates new table. Syntax: CREATE TABLE table_name (column1 datatype, ...)
ALTER TABLE Modifies existing table. Can ADD, MODIFY, or DROP columns.
DROP TABLE Deletes entire table including structure and data. Syntax: DROP TABLE table_name
Data Manipulation Language (DML) SQL commands to manipulate data: INSERT, UPDATE, DELETE, SELECT.
Term / Topic / Concept Explanation (Easy to Remember)
PL/SQL Procedural Language extension to SQL. Combines SQL with procedural features like loops, conditions, variables.
Term / Topic / Concept Explanation (Easy to Remember)
Transaction Logical unit of work with one or more operations. Either all execute or none (atomicity). Example: bank transfer (debit + credit).
